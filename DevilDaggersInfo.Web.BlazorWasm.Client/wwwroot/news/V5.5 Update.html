Global Custom Leaderboards

  • Added global custom leaderboards which combines every leaderboard in a category into a global leaderboard.
  • The global shows the amount of daggers per player, as well as a scoring system which is based on the daggers and ranking.

Custom Leaderboard Score API V2

  • Improved performance for score submissions.
  • Replay files are now validated upon submit.
  • Fixed saving incorrect homing stored value upon submit.
  • Fixed replay submissions being able to override highscore by a very small amount.
  • Fixed not always sorting scores by submit date when two scores had equal times.

Replay Parsing

  • Implemented new library that is able to parse (local) replay files, as well as build replay files from scratch. These features can be used for analyzing gameplay, or possibly for writing a TAS for a certain spawnset.
  • Most of the events have been figured out, but not all is known yet (unknown values are named A, B, C, etc).
  • The parsing feature can be tried out when viewing a custom leaderboard score. It parses the replay and displays all individual events on the page. See here for an example.
  • Maybe I'll make a new tool called DevilDaggersReplayEditor some time in the future, who knows...

Enhancements

  • Added menu item to view own player profile.
  • Dagger times for custom leaderboards can now be optional. These leaderboards are now referred to as "non-featured" rather than "archived".
  • Non-featured leaderboards are now displayed on the custom leaderboard overview page.
  • Non-featured leaderboards are excluded from the player page statistics.
  • Implemented new custom select list component which behaves correctly when reloading the page.
  • Implemented new custom checkbox component which behaves correctly when reloading the page.
  • Improved styling for all input UI.
  • Improved caching behavior for CSS/JS content.
  • New Speedrun leaderboards are now disallowed; use the Race category instead.
  • Descriptions for custom leaderboard categories have been added to the overview page.
  • Improved global statistics API with more data for accuracy stats.
  • Improved mobile page layout for spawnset, mod, and custom leaderboard overviews.
  • Added short descriptions to custom overview pages.
  • Client version is now displayed for custom leaderboard scores (hover over submit date).
  • Mods and spawsets are now ordered correctly on the player page.
  • Moved the "Download replay" button to the custom leaderboard score page itself.

  • Fixed incorrect caching behavior on Firefox.
